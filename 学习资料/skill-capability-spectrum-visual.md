### 4. 能力边界清单

**✅ Skills 擅长：**
```
✓ 重复性工作自动化
✓ 专业格式和标准执行
✓ 复杂流程的结构化指导
✓ 领域知识的系统化输出
✓ 多步骤工作流的协调
✓ 代码/文档/设计的生成
✓ 数据分析和模式识别
✓ 决策支持（提供选项和分析）
```

**❌ Skills 不擅长 / 不应该做：**
```
✗ 替代人类的最终判断
✗ 100% 无监督的自动决策
✗ 处理意外和极端情况
✗ 创造性突破（跳出框架）
✗ 模糊的伦理和法律决策
✗ 跨领域的深度推理（过于复杂）
✗ 实时响应关键系统故障
✗ 涉及重大财务/法律风险的操作
```

## 🎯 选择合适 Level 的决策树

```
开始: 我需要创建一个 Skill

                    ┌─── 只需要改变 Claude 的风格/语气？
                    │    └─> Level 0-1 (50 行以内)
                    │
                    ├─── 需要操作特定工具/文件格式？
                    │    └─> Level 2 (~150 行 + scripts)
你的需求是什么？────┤
                    ├─── 需要专业领域知识和判断？
                    │    └─> Level 3 (~300 行 + references)
                    │
                    ├─── 需要管理完整的工作流生命周期？
                    │    └─> Level 4 (~800 行 + 完整工具链)
                    │        ⚠️  确认你有维护能力
                    │
                    └─── 想要"AI 运营一切"？
                         └─> ❌ STOP!
                             ├─ 拆分成多个 Level 2-3 Skills
                             └─ 不要创建 Level 5

推荐策略：
┌─────────────────────────────────────────────┐
│  从小开始 → 验证价值 → 逐步扩展              │
│                                             │
│  示例路径：                                  │
```

## 💡 实践建议

### 对于初学者：
```
从 Level 1 开始练习：
1. 选择一个你熟悉的领域
2. 写一个 50 行的 SKILL.md
3. 测试、迭代、改进
4. 只在真正需要时添加 scripts/
```

### 对于有经验的用户：
```
瞄准 Level 2-3：
1. 识别重复性任务（每周 > 2 小时）
2. 设计 Level 2 skill（工具集成）
3. 如果需要深度知识，升级到 Level 3
4. 保持 SKILL.md 简洁（< 500 行）
```

### 对于企业团队：
```
构建 Skill 库：
1. 盘点公司特有的流程和知识
2. 为每个流程创建专注的 Skill
3. 优先创建 5-10 个 Level 2-3 Skills
4. 对关键业务流程，考虑 Level 4
5. 持续维护和更新
```

## 🎓 认知框架总结

**记住这个金字塔：**

```
                    ⚠️
                   ╱  ╲
                  ╱ L5 ╲  理论极限（不推荐）
                 ╱──────╲
                ╱   L4   ╲  企业系统（谨慎使用）
               ╱──────────╲
              ╱     L3     ╲  领域专家（高价值）
             ╱──────────────╲
            ╱       L2       ╲  工具集成（实用）
           ╱──────────────────╲
          ╱         L1         ╲  格式规范（简单）
         ╱──────────────────────╲
        ╱           L0           ╲  微调（入门）
       ╱──────────────────────────╲

      底座越宽 = 越多人使用
      顶端越尖 = 越少人需要
```

**核心原则：**
1. **简洁优先** - Claude 已经很聪明，只添加它不知道的
2. **按需加载** - 使用 references/ 和 assets/ 减少上下文压力
3. **专注胜于全能** - 10 个小 Skills > 1 个巨型 Skill
4. **人机协作** - Skills 是辅助，不是替代
